# Good Practices for Software Development

## Q.1 Which point(s) were new to you?
### The following points were new and insightful to me:

- Sharing written requirements with the team for quick feedback and future reference.
- Using screenshots, screencasts, and code-sharing tools to ask better questions.
- Joining meetings a few minutes early to build better relationships with teammates.
- Being mindful of teammates' time by sending all questions in a single message.
- Using productivity techniques such as Deep Work, blocking social media, and tracking time.

## Q.2 Which area do you think you need to improve on? What are your ideas to make progress in that area?
### I need to improve my communication and ask questions more effectively when I am stuck.

To make progress, I will:

- Clearly explain problems before asking for help.
- Share error messages, screenshots, and code snippets when needed.
- Communicate delays or blockers as soon as possible.
- Take detailed notes during requirement discussions.
- Stay focused during work hours by reducing distractions and managing my time better.

These improvements will help me collaborate better with my team and become more productive.
